利用Hexo部署个人静态网站到Github以及Coding上

Hexo

Hexo是一种高效的快速构建个人博客的博客框架。Hexo 利用的是Markdown语言进行博客的录入，通过简单几步搭建个人的博客。

GitHub

GitHub是利用Git作为唯一版本库格式进行托管的面向开源以及私有软件项目的托管平台，为了实现Hexo搭建个人网站，使用GitHub对个人博客的代码以及地址进行托管无疑是非常有利的。

Coding

Coding作为非常优秀的国内的代码托管仓库，在国内GitHub常常出现被墙掉以及访问缓慢的情况下，使用Coding进行网站地址的托管以便于国内的访问。

如何搭建个人网站

1.注册GitHub以及Coding账号

2.下载并安装Git Bush以及Node.js

下载地址：Git Bush | Node.js

  • Git Bush是利用Git进行操作命令的工具，安装包下载后打开一路点击下一步就可以了。
  • 之后安装Node.js，Hexo是基于nodeJS环境的静态博客，下载时建议选择LTS版本即可（长期支持的稳定版，Current版本是最新版）安装过程中在Custom Step选择 Add to PATH，其余一路下一步即可。

    3.命令安装Hexo

  • 选择一个个人网站存储的位置，新建一个文件夹命名为 blog （用于存储所有博客里的内容，当然其他英文名字都可以）
  • 使用git bush 或是cmd命令转移到该博客文件夹里 cd blog
  • 输入命令 npm install hexo-cli g 进行Hexo的安装
  • 输入命令 hexo init 初始化当前文件夹为Hexo搭建静态博客的位置
  • 输入命令 npm install 进行Hexo几本扩展插件的安装

  • repository name填写自己GitHub的用户名，然后选择public，点击create repository建立新的repo

7. 发博文

  • 新建一篇博文，使用cmd命令输入 hexo new post "title" 即可新建一篇题目为 title 的博文，内容的语法参考Markdown
  • 使用Atom即可方便快捷的访问每一篇博文，可以在 blog/source/_posts/ 中看到新建的 title.md 文件
  • 在cmd命令中运行 hexo server 启用本地预览，如遇到端口冲突等原因，可以运行 hexo server -p 4001 启用预览，预览地址为 http://localhost:4000 或是自定义的端口地址。
